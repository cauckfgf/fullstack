1、获取设备列表必须传，设备类型与设备型号，要获取所有设备，这个两个组合比较麻烦（轮询每个设备发一次http请求）
2、2.8与2.9接口获取最新数值只能一个设备一个设备获取，实际上需要一个获取所有设备（或者一种类型设备）的最新数据的接口
3、依据东方医院经验，客流返回参数需要一个时间段，查询也需要一个时间段
4、接口传id的字段是str，有些实例给的是整数，需要统一一下
5、报警阈值
6、2.8接口通过返回的key来区分传感器类型与名称,能否改成如下方式
{
	message:'',
	result:[
		{
			"name":"水位",
			"value":3.0,
			"unit":"升"
		},
		{
			"name":"ph值",
			"value":4.0,
			"unit":"pH"
		},
	],
	status:1,
	success:1
}
